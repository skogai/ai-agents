# OODA Loop

Decision-making framework for operating in uncertain, rapidly changing environments.

**O**bserve, **O**rient, **D**ecide, **A**ct

## Stages

| Stage | Action | Key Questions |
|-------|--------|---------------|
| Observe | Gather information | What's happening? What data do we have? |
| Orient | Analyze and synthesize | What does it mean? How does it fit context? |
| Decide | Choose a course of action | What should we do? What are trade-offs? |
| Act | Execute the decision | Implement quickly, then observe again |

## Key Insight

The loop is continuous and iterative. Faster OODA loops create competitive advantage. The goal is not perfect information but faster iteration.

## Pre-Mortem Application

The pre-mortem process maps directly to OODA stages. Use this mapping to ensure each phase produces actionable output.

| Pre-Mortem Phase | OODA Stage | Activity |
|------------------|------------|----------|
| Phase 1: Project Brief | Observe | Gather project scope, constraints, dependencies |
| Phase 2: Failure Announcement | Orient | Shift mindset from optimism to prospective hindsight |
| Phase 3: Independent Analysis | Orient | Analyze causes of imagined failure |
| Phase 4: Round-Robin Collection | Decide | Prioritize and categorize failure causes |
| Phase 5: Review and Mitigate | Act | Assign mitigations, owners, and review dates |

### Iterative Risk Review

Pre-mortems should not be one-time exercises. Schedule re-observation loops:

1. **Observe**: Review risk inventory against current project state
2. **Orient**: Identify new risks from changed scope, team, or timeline
3. **Decide**: Re-score existing risks, add new ones
4. **Act**: Update mitigations, reassign owners

### Anti-Patterns

| Anti-Pattern | Problem | Fix |
|--------------|---------|-----|
| Analysis paralysis in Phase 3 | Too much time analyzing, no action | Time-box to 5 minutes |
| Skipping Phase 5 | Risks identified but no mitigations | Always complete the full loop |
| One-and-done pre-mortem | No iteration as project evolves | Schedule reviews per OODA cadence |
| No re-observation | Risks become stale | Tie review schedule to project milestones |
